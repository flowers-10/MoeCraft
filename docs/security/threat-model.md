# MoeCraft 威胁模型

> 状态：G40 工程审查基线，2026-07-21。它不是法务意见，也不等于生产安全认证。

## 范围与资产

范围包括 storefront、admin、NestJS API、MySQL、迁移、日志/telemetry、文件元数据和 CI/release。主要资产是账户凭据与会话、商家资质、用户联系方式、商品/IP 资料、审计记录、订单/支付/库存事实和发布密钥。

信任边界：浏览器是不可信客户端；API 是业务规则边界；MySQL 只允许服务身份访问；对象存储、支付、通知和 telemetry 是外部 provider；CI 只使用临时测试凭据。任何由前端隐藏按钮、JWT 中自报角色或文件名推导出的权限都不可信。

## 风险登记

| 威胁 | 影响 | 当前控制 | 上线前证据/剩余动作 |
| --- | --- | --- | --- |
| 凭据填充、暴力登录 | 账号接管 | bcrypt、认证接口限流、统一错误、审计失败事件 | staging 用假账号验证限流；平台敏感账号 MFA |
| XSS 窃取浏览器令牌 | 冒用用户/商家身份 | 请求不回显 token，错误和日志脱敏 | admin 当前使用 sessionStorage；生产应迁移 HttpOnly、Secure、SameSite cookie 并配置 CSP |
| IDOR/跨商家读取 | 资质、商品或订单泄露 | AuthorizationGuard 读取数据库角色，商家 scope 断言，服务层按当前用户查询 | 每个新对象接口必须有跨商家测试；多商家会员模型落地前禁止共用账号 |
| 角色/按钮越权 | 改商品、审核或治理数据 | 角色权限、后台路由/按钮权限、API 端最终校验 | 角色矩阵见 `permission-review.md`；权限授予接口必须双重校验和审计 |
| SQL/命令注入 | 数据篡改或主机接管 | Prisma 参数化查询、DTO 白名单、无 shell 拼接输入 | 富文本、搜索、导出和后台筛选接入时追加 fuzz/滥用测试 |
| 恶意上传/私有文件泄露 | XSS、恶意文档、资质泄露 | MIME/大小校验、私有 object key、owner 元数据、隔离状态 | 签名 URL、病毒扫描、内容处置和下载审计必须随首个真实上传 provider 上线 |
| 重放/重复副作用 | 重复订单、扣款或退款 | 刷新令牌轮换；当前交易模块尚未实现 | 订单、支付、退款必须使用幂等键、唯一事件号和并发锁，未完成前不能生产收款 |
| PII/内部异常泄露 | 合规处罚、攻击情报 | 稳定错误码、安全 500、结构化日志字段脱敏、telemetry 不发请求体 | 审查 provider 保留期、访问审计、导出/注销和数据删除流程 |
| 依赖/密钥供应链 | 构建后门、凭据泄露 | frozen lockfile、依赖 audit、secret scan、不可变 digest 镜像 | 每次 release 保存扫描证据；密钥只由 CI/secret manager 注入 |
| 数据库故障/迁移破坏 | 全站不可用或数据丢失 | readiness 探针、迁移静态/Shadow gate、备份脚本、expand/contract 约束 | staging 真实恢复演练、RPO/RTO 记录和回滚负责人 |
| 日志/指标被滥用 | PII 侧漏或监控失真 | 无 Authorization/Cookie/密码记录，metrics 受 `system:manage` 保护 | collector HTTPS、最小权限、保留期和告警值班配置 |

## 可接受边界

G40 只登记风险，不把未实现的 provider、支付、订单、库存或售后能力标为已上线。任何高风险项必须有 owner、证据、到期日和回滚动作；没有证据的项目保持阻断状态。

生产放行最低条件：令牌存储方案完成评审、真实 staging 迁移/备份恢复通过、所有支付/退款写操作幂等、对象存储扫描可用、告警有人接听，且业务闭环任务 G16–G36 已完成相邻模块验收。
