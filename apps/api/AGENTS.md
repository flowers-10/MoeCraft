# API 协作指南

本目录为 NestJS API。

## 分层边界

- 控制器保持轻量：仅处理传输层，并将业务逻辑委托给服务。
- 新功能适用时应放入独立 Nest 模块，并包含控制器、服务和 DTO。
- 控制器不得直接访问持久化层；数据库集成应经由 `PrismaService` 等服务。
- 请求与响应契约必须明确；与客户端共用时，使用 `@moecraft/shared` 中的类型。
- 不得硬编码密钥、连接串或令牌。本地值放在未跟踪的 `apps/api/.env` 中。
- `/health` 必须保持快速且不依赖重型外部服务。

## 测试与验证

- API 改动后运行 `pnpm --filter @moecraft/api build`。
- 手动接口检查只能使用隔离的本地数据，绝不修改共享或生产数据库。
- 引入测试运行器后，先覆盖服务层业务规则，再覆盖控制器状态码和输入验证。除非任务明确提供依赖，不要编写需要外部服务的宽泛集成测试。
