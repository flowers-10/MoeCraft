# 商城前台协作指南

本目录为 Nuxt 3 客户端商城。

## UI 与数据规则

- 遵循 Nuxt 约定：`pages/` 使用文件路由，可复用 UI 放在 `components/`，可复用客户端逻辑引入后放在 composables。
- 商品、购物车和结算领域数据如被多端使用，应与 `@moecraft/shared` 的类型保持一致。
- 除正常状态外，同时设计加载、空数据、错误和无库存状态。
- 所有面向消费者的购买交互都必须在窄屏手机上可用。
- 客户端代码不得嵌入 API 密钥或任何高权限管理逻辑。

## 验证

前台改动后运行 `pnpm --filter @moecraft/storefront build`。涉及相关 UI 时，检查代表性的商品浏览流程和移动端布局。
