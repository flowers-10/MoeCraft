# 管理后台协作指南

本目录为 Vue 3 + Vite 运营后台。

## 目录职责

- `src/App.vue`：仅负责顶层页面组合和页面级状态。
- `src/components/`：存放职责单一、可复用的后台组件。
- `src/main.ts`：仅保留应用启动逻辑。

## UI 规则

- 使用 `<script setup lang="ts">`，所有 props、emits 和复杂数据结构必须类型化。
- 独立可复用的控件、面板、表格和导航必须拆为组件；不要让 `App.vue` 演变成单文件实现。
- API 客户端尚未接入时，模拟后台数据应放在对应展示组件附近，且不能将模拟数据表述为实时数据。
- 新布局必须同时设计桌面、平板与窄屏手机的响应式行为，明确采用溢出、堆叠、隐藏或滚动策略。
- 图标按钮应使用语义元素，并提供可读标签或 `aria-label`。
- 除非任务要求重设计，否则延续当前 MoeCraft 深色后台视觉风格。

## 验证

每次后台改动都运行 `pnpm --filter @moecraft/admin build`。交互控件需验证状态变化、事件派发和空状态。布局改动时，只有实际检查过桌面和移动端宽度后，才能声称已完成视觉验证。
