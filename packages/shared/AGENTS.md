# 共享包协作指南

本包管理跨应用共用的 TypeScript 契约与常量。

## 规则

- 仅放入至少两个应用会复用的代码，或客户端和服务端共用的 API 契约。
- 导出必须稳定、具名且与框架无关；不得引入 Vue、Nuxt、NestJS 或仅浏览器可用的 API。
- 优先使用明确的联合类型、接口、枚举/常量以及纯函数。
- 修改已导出的类型且具有破坏性时，视为跨包改动，并在同一任务中更新所有消费者。

## 验证

改动后运行 `pnpm --filter @moecraft/shared typecheck`，再对所有受影响的消费者运行类型检查。共享逻辑应保持确定性，以便无需框架即可进行单元测试。
